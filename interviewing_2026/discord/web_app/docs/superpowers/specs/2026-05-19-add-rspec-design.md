# Add RSpec to Rails backend

**Date:** 2026-05-19
**Scope:** `backend/` only. Frontend is untouched.

## Goal

Set up RSpec as the test framework for the Rails 8.1 API-only backend, with the common companion gems used in idiomatic Rails projects, and a working example spec for the only existing endpoint.

## Context

- `backend/` is a Rails 8.1 API-only app (`config.api_only = true`).
- No test framework is currently configured: there is no `test/` or `spec/` directory, and `require "rails/test_unit/railtie"` is commented out in `config/application.rb`.
- The only existing endpoint is `GET /api/hello` → `Api::HelloController#show`.

## Gems

Added to the `:development, :test` group in `backend/Gemfile`:

- `rspec-rails ~> 8.0` — framework + Rails integration
- `factory_bot_rails` — model/object factories
- `faker` — fake data for factories
- `shoulda-matchers ~> 6.0` — concise matchers for Rails

## Generated and edited files

- `backend/.rspec` — default `--require spec_helper` (generated by `rails g rspec:install`)
- `backend/spec/spec_helper.rb` — pure RSpec config (generated)
- `backend/spec/rails_helper.rb` — Rails-specific config (generated), then edited to:
  - Uncomment the `Dir[Rails.root.join('spec/support/**/*.rb')].sort.each { |f| require f }` line
  - Include `FactoryBot::Syntax::Methods` for all examples
  - Add a `Shoulda::Matchers.configure` block targeting `:rspec` with the `:rails` library
- `backend/spec/support/` — empty directory, ready to hold shared helpers
- `backend/spec/requests/api/hello_spec.rb` — working example request spec asserting `GET /api/hello` returns 200 with the expected JSON body
- `backend/.gitignore` — append `/spec/examples.txt` (rspec's run-persistence file)

## Conventions

- **Request specs** for controllers (Rails 8 + API-only idiomatic; controller specs are deprecated upstream).
- **Model specs** colocated under `spec/models/` once models exist.
- **Factories** under `spec/factories/`.
- Run the suite with `bundle exec rspec` from `backend/`. No Rake glue required — `rspec-rails` wires `rake spec` automatically.

## Out of scope

- CI integration (GitHub Actions, etc.)
- Coverage reporting (SimpleCov)
- HTTP mocking (VCR, WebMock)
- Parallel test runners
- System / browser specs

Each is easy to layer on later when the need is concrete.

## Success criteria

1. `cd backend && bundle install` succeeds.
2. `cd backend && bundle exec rspec` runs the example request spec and reports 1 example, 0 failures.
3. `spec/rails_helper.rb` requires `factory_bot_rails` and `shoulda-matchers` without errors.
