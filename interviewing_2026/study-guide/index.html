    Universal solving framework (say it out loud, every time)

    1. Restate the problem in your own words.
    2. Clarify 2–3 things: input size & ranges, duplicates?, sorted?, can I mutate?, what to return on empty / no-solution?
    3. Examples: walk the given one by hand; invent one tiny edge case (empty, single, all-equal).
    4. Brute force first, out loud, with its complexity — even if you won't code it. ("Naive is O(n²) because…")
    5. Name the pattern and the better complexity you're targeting. Get a nod before coding.
    6. Code, narrating only non-obvious decisions (skip "now a for loop").
    7. Trace your code on the concrete example.
    8. State complexity (time + space) unprompted, then ask: "Want me to optimize, add tests, or handle more edges?"

    The single biggest score lever is narrating the brute-force → optimized transition and why each optimization helps. Interviewers buy reasoning, not just the final code.

    • Recursion uses stack space = max depth (tree height, n for a list/chain).

    Arrays & Hashing trade space for time — the most common interview move

    The mental model: a hash map turns "have I seen / how many / where is" questions from an O(n) inner scan into an O(1) lookup. Prefix sums turn "sum of a range" into a subtraction. Reach here first when you catch yourself writing a nested loop over the same array.

    Two Sum ✓ Reviewed

    Prompt: Given nums and a target, return indices of the two numbers that add to target. Exactly one answer; can't reuse an element. (LC 1)

    Strategy: One pass. For each number, the partner you need is target - num. Keep a hash of value => index of everything seen so far; check for the partner before inserting the current number so you never pair an element with itself.

    Complexity: Time O(n) · Space O(n). (Brute force is O(n²) — say that first, then improve.)

    Ruby

    # Input:  nums = [2, 7, 11, 15], target = 9
# Output: [0, 1]   (because nums[0] + nums[1] == 9)

def two_sum(nums, target)
  seen = {}                       # value => index we saw it at
  nums.each_with_index do |num, i|
    complement = target - num     # the partner that would complete the pair
    # If we've already seen the partner, we're done. Check BEFORE inserting
    # so a single element can't be used twice.
    return [seen[complement], i] if seen.key?(complement)
    seen[num] = i                 # remember this number for future partners
  end
  []                              # problem guarantees a solution; defensive return
end

p two_sum([2, 7, 11, 15], 9)     # => [0, 1]
p two_sum([3, 2, 4], 6)          # => [1, 2]

    Python

def two_sum(nums, target):
    seen = {}                          # value -> index we saw it at
    for i, num in enumerate(nums):
        complement = target - num      # partner that completes the pair
        if complement in seen:         # check BEFORE inserting current num
            return [seen[complement], i]
        seen[num] = i                  # remember for future partners
    return []                          # guaranteed solution; defensive

    Group Anagrams New

    Prompt: Given a list of strings, group the ones that are anagrams of each other. (LC 49)

    Strategy: Two strings are anagrams iff they share a canonical key. Cheapest correct key: the sorted characters ("eat" -> "aet"). Bucket every word under its key in a hash of key => [words]. (Faster key: a 26-length count tuple — O(n·k) instead of O(n·k log k) — mention it.)

    Complexity: Time O(n · k log k) for n words of length k · Space O(n · k).

    Ruby

    # Input:  ["eat","tea","tan","ate","nat","bat"]
# Output: [["eat","tea","ate"], ["tan","nat"], ["bat"]]

def group_anagrams(words)
  # Hash whose missing keys auto-initialize to [] so we can push freely.
  groups = Hash.new { |h, k| h[k] = [] }
  words.each do |word|
    key = word.chars.sort.join      # "tea" -> "aet" — same for all anagrams
    groups[key] << word
  end
  groups.values                     # we only care about the grouped lists
end

p group_anagrams(["eat","tea","tan","ate","nat","bat"])

    Python

    # Input:  ["eat","tea","tan","ate","nat","bat"]
# Output: [["eat","tea","ate"], ["tan","nat"], ["bat"]]

from collections import defaultdict

def group_anagrams(words):
    groups = defaultdict(list)           # missing key -> [] automatically
    for word in words:
        key = "".join(sorted(word))      # "tea" -> "aet", shared by anagrams
        groups[key].append(word)
    return list(groups.values())

    Top K Frequent Elements ✓ Reviewed

    Prompt: Return the k most frequent elements in nums. (LC 347)

    Strategy: Count with a hash. Then two good options: (a) bucket sort by frequency — index = frequency, value = list of numbers with that frequency — and walk buckets from high to low for O(n); or (b) a size-k min-heap for O(n log k). Bucket sort is the optimal answer; state both.

    Complexity: Bucket O(n) time / O(n) space · Heap O(n log k) time / O(k) space.

    Ruby

    # Input:  nums = [1,1,1,2,2,3], k = 2
# Output: [1, 2]   (order among ties not required)

def top_k_frequent(nums, k)
  counts = Hash.new(0)
  nums.each { |n| counts[n] += 1 }     # value => frequency

  # Bucket sort: buckets[f] holds all numbers seen exactly f times.
  # Max possible frequency is nums.length, so size that many + 1.
  buckets = Array.new(nums.length + 1) { [] }
  counts.each { |num, freq| buckets[freq] << num }

  result = []
  # Walk from highest frequency down until we've collected k numbers.
  buckets.length.downto(0) do |freq|
    buckets[freq].each do |num|
      result << num
      return result if result.length == k
    end
  end
  result
end

p top_k_frequent([1,1,1,2,2,3], 2)    # => [1, 2]

    Python

    # Input:  nums = [1,1,1,2,2,3], k = 2
# Output: [1, 2]

from collections import Counter

def top_k_frequent(nums, k):
    counts = Counter(nums)                 # value -> frequency

    # buckets[f] holds all numbers that occur exactly f times.
    buckets = [[] for _ in range(len(nums) + 1)]
    for num, freq in counts.items():
        buckets[freq].append(num)

    result = []
    for freq in range(len(nums), 0, -1):   # high frequency -> low
        for num in buckets[freq]:
            result.append(num)
            if len(result) == k:
                return result
    return result

print(top_k_frequent([1,1,1,2,2,3], 2))    # -> [1, 2]

    Subarray Sum Equals K ✓ Reviewed

    Prompt: Count the number of contiguous subarrays whose sum equals k. Values may be negative. (LC 560)

    Strategy: The classic prefix-sum + hash. Let prefix = sum of everything so far. A subarray ending here sums to k exactly when some earlier prefix equals prefix - k. Keep a hash of prefix_value => how many times seen; add that count. Seed with {0 => 1} so a subarray starting at index 0 is counted. (Sliding window does NOT work — negatives break monotonicity. Say this.)

    Complexity: Time O(n) · Space O(n).

    Ruby

    # Input:  nums = [1, 1, 1], k = 2
# Output: 2   (subarrays [1,1] at indices 0-1 and 1-2)

def subarray_sum(nums, k)
  count  = 0
  prefix = 0
  seen   = Hash.new(0)
  seen[0] = 1                       # empty prefix: enables subarrays from index 0
  nums.each do |num|
    prefix += num                   # running sum of nums[0..i]
    # If (prefix - k) was a prefix we saw before, every such occurrence marks
    # the start of a subarray ending here that sums to k.
    count += seen[prefix - k]
    seen[prefix] += 1               # record this prefix for future indices
  end
  count
end

p subarray_sum([1, 1, 1], 2)       # => 2
p subarray_sum([1, 2, 3], 3)       # => 2  ([1,2] and [3])

    Python

    # Input:  nums = [1, 1, 1], k = 2
# Output: 2

from collections import defaultdict

def subarray_sum(nums, k):
    count = 0
    prefix = 0
    seen = defaultdict(int)
    seen[0] = 1                       # empty prefix: subarrays starting at 0
    for num in nums:
        prefix += num                 # running sum
        count += seen[prefix - k]     # earlier prefixes that close a k-sum here
        seen[prefix] += 1
    return count

    Product of Array Except Self New

    Prompt: Return an array where out[i] is the product of all elements except nums[i]. No division allowed; O(n) time. (LC 238)

    Strategy: out[i] = (product of everything left of i) × (product of everything right of i). Do a left-to-right pass filling prefix products, then a right-to-left pass multiplying in suffix products with a running variable. The output array doubles as scratch space → O(1) extra.

    Complexity: Time O(n) · Space O(1) extra (output aside).

    Ruby

    # Input:  nums = [1, 2, 3, 4]
# Output: [24, 12, 8, 6]

def product_except_self(nums)
  n = nums.length
  out = Array.new(n, 1)

  # Pass 1 (left -> right): out[i] = product of everything strictly left of i.
  prefix = 1
  (0...n).each do |i|
    out[i] = prefix
    prefix *= nums[i]               # extend the running left-product
  end

  # Pass 2 (right -> left): multiply in the product of everything to the right.
  suffix = 1
  (n - 1).downto(0) do |i|
    out[i] *= suffix
    suffix *= nums[i]               # extend the running right-product
  end
  out
end

p product_except_self([1, 2, 3, 4]) # => [24, 12, 8, 6]

    Python

    # Input:  nums = [1, 2, 3, 4]
# Output: [24, 12, 8, 6]

def product_except_self(nums):
    n = len(nums)
    out = [1] * n

    prefix = 1                        # product of everything strictly left of i
    for i in range(n):
        out[i] = prefix
        prefix *= nums[i]

    suffix = 1                        # product of everything strictly right of i
    for i in range(n - 1, -1, -1):
        out[i] *= suffix
        suffix *= nums[i]
    return out

print(product_except_self([1, 2, 3, 4]))  # -> [24, 12, 8, 6]

    Two Pointers & Sliding Window turn O(n²) scans into O(n) sweeps

    Two pointers: on sorted data, move a left/right pair inward based on a comparison. Sliding window: for "longest/shortest contiguous run with property P" — expand right to include, shrink left while the window is invalid. The window's contents are usually tracked in a hash/counter.

    3Sum ✓ Reviewed

    Prompt: Return all unique triplets [a, b, c] in nums with a + b + c == 0. (LC 15)

    Strategy: Sort. Fix each i; the rest reduces to "two-sum on a sorted array" using two pointers lo/hi moving inward: sum too small → lo++, too big → hi--, equal → record. The trick is skipping duplicates at i, lo, and hi to keep triplets unique.

    Complexity: Time O(n²) · Space O(1) extra (ignoring output / sort stack).

    Ruby

    # Input:  nums = [-1, 0, 1, 2, -1, -4]
# Output: [[-1, -1, 2], [-1, 0, 1]]

def three_sum(nums)
  nums = nums.sort                 # enables two-pointer + easy dup-skipping
  result = []
  n = nums.length
  (0...n).each do |i|
    next if i > 0 && nums[i] == nums[i - 1]   # skip duplicate anchor values
    break if nums[i] > 0                       # sorted: no way to reach 0 anymore
    lo, hi = i + 1, n - 1
    while lo < hi
      total = nums[i] + nums[lo] + nums[hi]
      if total < 0
        lo += 1                    # need a bigger sum
      elsif total > 0
        hi -= 1                    # need a smaller sum
      else
        result << [nums[i], nums[lo], nums[hi]]
        lo += 1; hi -= 1
        lo += 1 while lo < hi && nums[lo] == nums[lo - 1]  # skip dup lefts
        hi -= 1 while lo < hi && nums[hi] == nums[hi + 1]  # skip dup rights
      end
    end
  end
  result
end

p three_sum([-1, 0, 1, 2, -1, -4])   # => [[-1, -1, 2], [-1, 0, 1]]

    Python

    # Input:  nums = [-1, 0, 1, 2, -1, -4]
# Output: [[-1, -1, 2], [-1, 0, 1]]

def three_sum(nums):
    nums.sort()
    result = []
    n = len(nums)
    for i in range(n):
        if i > 0 and nums[i] == nums[i - 1]:   # skip duplicate anchor
            continue
        if nums[i] > 0:                        # sorted: can't sum to 0 anymore
            break
        lo, hi = i + 1, n - 1
        while lo < hi:
            total = nums[i] + nums[lo] + nums[hi]
            if total < 0:
                lo += 1
            elif total > 0:
                hi -= 1
            else:
                result.append([nums[i], nums[lo], nums[hi]])
                lo += 1
                hi -= 1
                while lo < hi and nums[lo] == nums[lo - 1]:  # skip dup lefts
                    lo += 1
                while lo < hi and nums[hi] == nums[hi + 1]:  # skip dup rights
                    hi -= 1
    return result

print(three_sum([-1, 0, 1, 2, -1, -4]))   # -> [[-1, -1, 2], [-1, 0, 1]]

    Container With Most Water New

    Prompt: Each height[i] is a vertical line. Pick two lines that with the x-axis form a container holding the most water. (LC 11)

    Strategy: Two pointers at the ends. Area = min(left, right) × width. The width only shrinks as pointers move in, so to have any hope of a bigger area you must raise the limiting (shorter) wall — always move the pointer at the shorter line. Greedy, provably optimal.

    Complexity: Time O(n) · Space O(1).

    Ruby

    # Input:  height = [1,8,6,2,5,4,8,3,7]
# Output: 49

def max_area(height)
  lo, hi = 0, height.length - 1
  best = 0
  while lo < hi
    width = hi - lo
    h = [height[lo], height[hi]].min   # water is capped by the shorter wall
    best = [best, width * h].max
    # Moving the taller wall can never help (width drops, height capped by
    # the short one). Move the shorter wall to look for a taller bound.
    if height[lo] < height[hi]
      lo += 1
    else
      hi -= 1
    end
  end
  best
end

p max_area([1,8,6,2,5,4,8,3,7])    # => 49

    Longest Substring Without Repeating Characters ✓ Reviewed

    Prompt: Length of the longest substring of s with all distinct characters. (LC 3)

    Strategy: Sliding window with a hash of char => last index seen. Expand right. When the current char was seen at or after left, jump left to just past its previous position (no need to step one at a time). Track the max width.

    Complexity: Time O(n) · Space O(min(n, alphabet)).

    Ruby

    # Input:  s = "abcabcbb"
# Output: 3   ("abc")

def length_of_longest_substring(s)
  last = {}                        # char => most recent index
  left = 0
  best = 0
  s.chars.each_with_index do |ch, right|
    # If we've seen ch inside the current window, shrink from the left to
    # just past its last occurrence so the window stays duplicate-free.
    if last.key?(ch) && last[ch] >= left
      left = last[ch] + 1
    end
    last[ch] = right
    best = [best, right - left + 1].max
  end
  best
end

p length_of_longest_substring("bbbbb")      # => 1

    Longest Repeating Character Replacement New

    Prompt: You may replace at most k characters in s. Return the length of the longest substring of a single repeated letter you can make. (LC 424)

    Strategy: Sliding window with a count of letters in the window. A window is valid when window_size - count_of_most_frequent_letter <= k (the non-majority letters are the ones we'd replace). Grow right; if invalid, slide left forward once (the window never needs to shrink below the best seen). Track max window size.

    Complexity: Time O(n) · Space O(1) (26 letters).

    Ruby

    # Input:  s = "AABABBA", k = 1
# Output: 4   (replace one B in "AABA"->? -> "AAAA"-ish window of length 4)

def character_replacement(s, k)
  counts = Hash.new(0)
  left = 0
  max_freq = 0                     # highest single-letter count in the window
  best = 0
  s.chars.each_with_index do |ch, right|
    counts[ch] += 1
    max_freq = [max_freq, counts[ch]].max
    # Replacements needed = window length - most common letter's count.
    # If that exceeds k, the window is invalid -> slide left by one.
    if (right - left + 1) - max_freq > k
      counts[s[left]] -= 1
      left += 1
    end
    best = [best, right - left + 1].max
  end
  best
end

    Python

    # Input:  s = "AABABBA", k = 1

from collections import defaultdict

def character_replacement(s, k):
    counts = defaultdict(int)
    left = 0
    max_freq = 0                       # most common letter's count in window
    best = 0
    for right, ch in enumerate(s):
        counts[ch] += 1
        max_freq = max(max_freq, counts[ch])
        if (right - left + 1) - max_freq > k:   # too many to replace
            counts[s[left]] -= 1
            left += 1
        best = max(best, right - left + 1)
    return best

    Minimum Window Substring ✓ Reviewed

    Prompt: Smallest substring of s that contains every character of t (with multiplicity). Return "" if none. (LC 76 · hard)

    Strategy: Sliding window with a "need" counter (chars required from t) and a have/required match count. Expand right; whenever the window is complete, record it and shrink left aggressively to find the minimum, decrementing matches as you drop chars. The have == required check is the heart of it.

    Complexity: Time O(n + m) · Space O(unique chars in t).

    Ruby

    # Input:  s = "ADOBECODEBANC", t = "ABC"
# Output: "BANC"

def min_window(s, t)
  return "" if t.empty? || s.length < t.length
  need = Hash.new(0)
  t.each_char { |c| need[c] += 1 }
  required = need.size             # distinct chars we must fully satisfy

  window = Hash.new(0)
  have = 0                         # how many distinct chars are fully satisfied
  best_len = Float::INFINITY
  best_start = 0
  left = 0

  s.chars.each_with_index do |ch, right|
    window[ch] += 1
    have += 1 if need.key?(ch) && window[ch] == need[ch]

    # While the window has everything, try to shrink it from the left.
    while have == required
      if (right - left + 1) < best_len
        best_len = right - left + 1
        best_start = left
      end
      drop = s[left]
      window[drop] -= 1
      have -= 1 if need.key?(drop) && window[drop] < need[drop]
      left += 1
    end
  end

  best_len == Float::INFINITY ? "" : s[best_start, best_len]
end

def min_window(s, t):
    if not t or len(s) < len(t):
        return ""
    need = Counter(t)
    required = len(need)              # distinct chars to satisfy

    window = defaultdict(int)
    have = 0
    best_len = float('inf')
    best_start = 0
    left = 0

    for right, ch in enumerate(s):
        window[ch] += 1
        if ch in need and window[ch] == need[ch]:
            have += 1

        while have == required:       # window complete -> shrink to minimize
            if right - left + 1 < best_len:
                best_len = right - left + 1
                best_start = left
            drop = s[left]
            window[drop] -= 1
            if drop in need and window[drop] < need[drop]:
                have -= 1
            left += 1

    return "" if best_len == float('inf') else s[best_start:best_start + best_len]

    Binary Search ★ gap-filler — the "halve the search space" pattern

    When: the input is sorted, OR you can phrase the answer as "the smallest/largest value X for which a yes/no predicate feasible(X) flips from false to true." That second form — binary search on the answer — is the senior move most people miss. The whole family is about maintaining a half-open invariant and never going infinite.

    Template + invariants (read this first)

    • Use lo, hi and loop while lo <= hi for "find exact value"; loop while lo < hi for "find boundary / leftmost." Pick one convention and stick to it.
    • Compute mid = lo + (hi - lo) / 2 — avoids overflow in other languages and reads as intent.
    • Every branch must shrink the range (move lo or hi past mid), or you loop forever. This is the #1 binary-search bug.
    • For "search on answer": define feasible(x) monotonic (false…false, true…true), then binary-search the smallest x with feasible(x) == true.

    Classic Binary Search New

    Prompt: Return the index of target in a sorted array, or -1. (LC 704)

    Strategy: The canonical exact-match template with lo <= hi. Compare mid to target and discard the impossible half.

    Complexity: Time O(log n) · Space O(1).

    Ruby

    # Input:  nums = [-1, 0, 3, 5, 9, 12], target = 9
# Output: 4

def binary_search(nums, target)
  lo, hi = 0, nums.length - 1      # inclusive range [lo, hi]
  while lo <= hi
    mid = lo + (hi - lo) / 2        # midpoint without overflow
    if nums[mid] == target
      return mid
    elsif nums[mid] < target
      lo = mid + 1                  # target is in the right half
    else
      hi = mid - 1                  # target is in the left half
    end
  end
  -1                               # range collapsed: not present
end

    Complexity: Time O(log n) · Space O(1).

    if nums[mid] < x
      lo = mid + 1                  # mid too small; answer is to the right
    else
      hi = mid                     # mid could be the boundary; keep it in range
    end
  end
  lo                               # first index not less than x (may == length)
end

  return [-1, -1] if start == nums.length || nums[start] != target
  last = lower_bound(nums, target + 1) - 1   # one before first value > target

    Search in Rotated Sorted Array New skill

    Prompt: Find target's index in a rotated sorted array (distinct values), or -1, in O(log n). (LC 33)

    Strategy: At each step one half is always properly sorted. Detect which (compare nums[lo] to nums[mid]). If the sorted half contains the target (range check), search it; otherwise search the other half. This "which half is clean, is target in it" reasoning is the whole trick.

    Complexity: Time O(log n) · Space O(1).

    Ruby

    # Input:  nums = [4, 5, 6, 7, 0, 1, 2], target = 0
# Output: 4

def search_rotated(nums, target)
  lo, hi = 0, nums.length - 1
  while lo <= hi
    mid = lo + (hi - lo) / 2
    return mid if nums[mid] == target

    if nums[lo] <= nums[mid]        # LEFT half [lo..mid] is sorted
      if nums[lo] <= target && target < nums[mid]
        hi = mid - 1               # target lies inside the sorted left half
      else
        lo = mid + 1               # otherwise it's in the right half
      end
    else                           # RIGHT half [mid..hi] is sorted
      if nums[mid] < target && target <= nums[hi]
        lo = mid + 1               # target lies inside the sorted right half
      else
        hi = mid - 1
      end
    end
  end
  -1
end

    Koko Eating Bananas New skill

    Prompt: Piles of bananas; Koko eats at speed k per hour (one pile at a time, leftovers wait). Find the minimum k to finish all piles within h hours. (LC 875)

    Strategy: The textbook binary search on the answer. Speed is between 1 and max(piles). feasible(k) = "can finish in ≤ h hours at speed k" is monotonic (faster is always ≥ as feasible).     Complexity: Time O(n · log(max pile)) · Space O(1).

def min_eating_speed(piles, h)
  # hours needed to clear all piles at speed k (ceil division per pile)
  hours_at = ->(k) { piles.sum { |p| (p + k - 1) / k } }

  lo, hi = 1, piles.max            # speed range [1, biggest pile]
  while lo < hi                    # searching for the leftmost feasible speed
    mid = lo + (hi - lo) / 2
    if hours_at.call(mid) <= h
      hi = mid                     # mid works; maybe a smaller speed also works
    else
      lo = mid + 1                 # too slow; need a faster speed
    end
  end
  lo                               # smallest k with hours_at(k) <= h
end

    Valid Parentheses ✓ Reviewed

    Prompt: Given a string of ()[]{}, decide if every bracket is closed by the right type in the right order. (LC 20)

    Strategy: Push opening brackets. On a closer, the top of the stack must be its matching opener — else invalid. Map closers→openers for a clean check. Valid iff the stack is empty at the end.

    Complexity: Time O(n) · Space O(n).

    Ruby

    # Input:  s = "([)]"  -> false ;  s = "()[]{}" -> true

def valid_parentheses(s)
  pairs = { ")" => "(", "]" => "[", "}" => "{" }   # closer => its opener
  stack = []
  s.each_char do |ch|
    if pairs.key?(ch)              # it's a closer
      # Must match the most recently opened bracket.
      return false if stack.empty? || stack.pop != pairs[ch]
    else
      stack.push(ch)               # opener: remember it
    end
  end
  stack.empty?                     # leftover openers => invalid
end

    Evaluate Reverse Polish Notation New

    Prompt: Evaluate an arithmetic expression in RPN (postfix), e.g. ["2","1","+","3","*"] = 9. (LC 150)

    Strategy: Push numbers. On an operator, pop the top two operands, apply, push the result. Watch operand order for - and / (second-popped is the left operand), and truncate division toward zero.

    Complexity: Time O(n) · Space O(n).

    Ruby

    # Input:  ["2","1","+","3","*"]  -> 9 ;  ["4","13","5","/","+"] -> 6

def eval_rpn(tokens)
  ops = %w[+ - * /]
  stack = []
  tokens.each do |tok|
    if ops.include?(tok)
      b = stack.pop                # second operand (right)
      a = stack.pop                # first operand (left)
      result =
        case tok
        when "+" then a + b
        when "-" then a - b
        when "*" then a * b
        when "/" then (a.to_f / b).truncate   # truncate toward zero
        end
      stack.push(result.to_i)
    else
      stack.push(tok.to_i)         # it's a number
    end
  end
  stack.last
end

    Min Stack New

    Prompt: Design a stack supporting push, pop, top, and get_min all in O(1). (LC 155)

    Strategy: Keep a second "mins" stack in lockstep: each entry is the minimum of everything at or below it. On push, store min(new, current_min); pop both together. get_min is just the top of the mins stack.

    Complexity: All ops O(1) · Space O(n).

    Ruby

    class MinStack
  def initialize
    @stack = []
    @mins  = []                    # @mins[i] = min of @stack[0..i]
  end

  def push(val)
    @stack.push(val)
    # New running minimum is val or the previous minimum, whichever is smaller.
    current_min = @mins.empty? ? val : [val, @mins.last].min
    @mins.push(current_min)
  end

  def pop
    @mins.pop                      # keep the two stacks in lockstep
    @stack.pop
  end

  def top
    @stack.last
  end

  def get_min
    @mins.last                     # O(1) minimum
  end
end

  stack = []                       # indices of days awaiting a warmer day

      answer[prev] = i - prev      # number of days waited
    end
    stack.push(i)                  # today now waits for its warmer day
  end

  while curr

    curr.next = prev               # flip this node's pointer backward
    prev = curr                    # advance prev and curr forward
    curr = nxt
  end
  prev                             # new head (old tail)

    Prompt: Is the tree a valid BST? (Every left descendant < node < every right descendant — not just direct children.) (LC 98)

def valid_bst?(node, low = -Float::INFINITY, high = Float::INFINITY)
  return true if node.nil?                       # empty subtree is valid
  return false unless node.val > low && node.val < high  # must fit open interval

    valid_bst?(node.right, node.val, high)
end

  return root if root == p || root == q

  right = lowest_common_ancestor(root.right, p, q)

  return root if left && right

    Binary Tree Level Order Traversal ✓ Reviewed

    Prompt: Return node values grouped by level, top to bottom. (LC 102)

    Strategy: BFS with a queue. The key idea: at the start of each outer loop, snapshot the current queue size — that's exactly one level's worth of nodes. Process that many, enqueuing their children for the next level.

    Complexity: Time O(n) · Space O(n).

    Ruby

    # Output for tree [3,[9],[20,[15],[7]]]: [[3], [9, 20], [15, 7]]

def level_order(root)
  return [] if root.nil?
  result = []
  queue = [root]                   # use index/rotation to avoid O(n) shift
  until queue.empty?
    level_size = queue.length      # snapshot: how many nodes on this level
    level = []
    level_size.times do
      node = queue.shift           # (fine for clarity; deque-style for perf)
      level << node.val
      queue << node.left  if node.left
      queue << node.right if node.right
    end
    result << level
  end
  result
end

    Heaps & Intervals ★ gap-filler — "best-so-far" and "overlapping ranges"

    Heap signals: "k largest/smallest," "median of a stream," "merge k sorted." A heap gives you the current min/max in O(1) and reshuffles in O(log n). Interval signals: ranges that might overlap — almost always sort by start, then sweep; counting concurrent intervals is a min-heap of end times. Python has heapq (min-heap); Ruby has none, so use the helper below or a sort.

    Ruby heap helper (Python users skip — use heapq)

    Why: Ruby ships no priority queue. Memorize this ~25-line binary heap; it's min-heap by default and takes a comparator block for max-heaps or custom keys. (In Python, import heapq and push tuples; for a max-heap push negated keys.)

    Ruby

    class Heap
  # Pass a block returning -1/0/1; default is a MIN-heap on the element itself.
  def initialize(&cmp)
    @d = []
    @cmp = cmp || ->(a, b) { a <=> b }
  end
  def size;   @d.size;   end
  def empty?; @d.empty?; end
  def peek;   @d[0];     end

  def push(x)
    @d << x
    i = @d.size - 1
    while i > 0                          # sift up while smaller than parent
      par = (i - 1) / 2
      break if @cmp.call(@d[i], @d[par]) >= 0
      @d[i], @d[par] = @d[par], @d[i]
      i = par
    end
  end

  def pop
    top = @d[0]
    last = @d.pop
    unless @d.empty?
      @d[0] = last; i = 0; n = @d.size
      loop do                            # sift down to the smaller child
        l, r, best = 2*i + 1, 2*i + 2, i
        best = l if l < n && @cmp.call(@d[l], @d[best]) < 0
        best = r if r < n && @cmp.call(@d[r], @d[best]) < 0
        break if best == i
        @d[i], @d[best] = @d[best], @d[i]; i = best
      end
    end
    top
  end
end

def merge_intervals(intervals)
  intervals = intervals.sort_by { |start, _| start }   # sort by start time
  merged = []
  intervals.each do |start, finish|
    if merged.any? && start <= merged.last[1]
      # Overlaps the last merged interval -> extend its end.
      merged.last[1] = [merged.last[1], finish].max
    else
      merged << [start, finish]     # disjoint -> start a new interval
    end
  end
  merged

    Find Median from Data Stream New skill

    Prompt: Support add_num(x) and find_median() over a growing stream. (LC 295 · hard)

    Strategy: The classic two-heap balance. A max-heap holds the smaller half, a min-heap the larger half. Keep their sizes within 1. The median is the max-heap's top (odd total) or the average of both tops (even). Add then rebalance so every element in lo ≤ every element in hi.

    Complexity: add O(log n) · median O(1) · Space O(n).

    Ruby

    class MedianFinder
  def initialize
    @lo = Heap.new { |a, b| b <=> a }   # MAX-heap: smaller half
    @hi = Heap.new                       # MIN-heap: larger half
  end

  def add_num(num)
    @lo.push(num)                  # always push to lo first...
    @hi.push(@lo.pop)              # ...then move lo's max over to hi
    # Rebalance so lo is never more than one element bigger than hi.
    @lo.push(@hi.pop) if @hi.size > @lo.size
  end

  def find_median
    if @lo.size > @hi.size
      @lo.peek.to_f                # odd count: middle lives in lo
    else
      (@lo.peek + @hi.peek) / 2.0  # even count: average the two middles
    end
  end
end

def num_islands(grid)
  return 0 if grid.empty?
  rows, cols = grid.length, grid[0].length

  sink = lambda do |r, c|
    # Stop at the border or on water/visited cells.
    return if r < 0 || c < 0 || r >= rows || c >= cols || grid[r][c] != "1"
    grid[r][c] = "0"               # mark visited by sinking the land
    sink.call(r + 1, c); sink.call(r - 1, c)   # flood all 4 neighbours
    sink.call(r, c + 1); sink.call(r, c - 1)
  end

  count = 0
  rows.times do |r|
    cols.times do |c|
      if grid[r][c] == "1"
        count += 1                 # new landmass discovered
        sink.call(r, c)            # erase it so we don't recount
      end
    end
  end
  count
end

    Rotting Oranges (multi-source BFS) New skill

    Prompt: Grid of 0 (empty), 1 (fresh), 2 (rotten). Each minute, rotten oranges rot 4-adjacent fresh ones. Return minutes until none are fresh, or -1 if impossible. (LC 994)

    Strategy: Multi-source BFS: seed the queue with all rotten oranges at once, then expand level by level — each BFS level is one minute. Track fresh count; if any remain when the queue empties, return -1. This "start from all sources simultaneously" trick is worth internalizing.

    Complexity: Time O(rows·cols) · Space O(rows·cols).

    Ruby

    # Input:  [[2,1,1],[1,1,0],[0,1,1]]   Output: 4

def oranges_rotting(grid)
  rows, cols = grid.length, grid[0].length
  queue = []                       # all currently-rotten cells (BFS sources)
  fresh = 0
  rows.times do |r|
    cols.times do |c|
      queue << [r, c] if grid[r][c] == 2
      fresh += 1 if grid[r][c] == 1
    end
  end

  minutes = 0
  dirs = [[1,0],[-1,0],[0,1],[0,-1]]
  read = 0                          # index-based queue read (avoid O(n) shift)
  while read < queue.length && fresh > 0
    minutes += 1
    level_end = queue.length        # snapshot this minute's rotten cells
    while read < level_end
      r, c = queue[read]; read += 1
      dirs.each do |dr, dc|
        nr, nc = r + dr, c + dc
        next if nr < 0 || nc < 0 || nr >= rows || nc >= cols || grid[nr][nc] != 1
        grid[nr][nc] = 2            # this fresh orange rots now
        fresh -= 1
        queue << [nr, nc]           # it spreads next minute
      end
    end
  end

  fresh == 0 ? minutes : -1         # leftover fresh => unreachable
end

        for c in range(cols):
            if grid[r][c] == 2:
                queue.append((r, c))
            elif grid[r][c] == 1:
                fresh += 1

    Course Schedule (topological sort) New skill

    Prompt: numCourses courses, prerequisites [a, b] meaning b before a. Can you finish all? (Equivalent: is the dependency graph acyclic?) (LC 207)

    Strategy — Kahn's algorithm (BFS): compute each course's in-degree (number of prereqs). Queue all zero-in-degree courses; repeatedly take one, "finish" it, and decrement its dependents' in-degrees, enqueuing any that hit zero. If you process all courses, no cycle; if some never reach zero, there's a cycle → impossible.

    Complexity: Time O(V + E) · Space O(V + E).

    Ruby

    # Input:  num=2, prereqs=[[1,0]]      -> true
#         num=2, prereqs=[[1,0],[0,1]] -> false (cycle)

def can_finish(num_courses, prerequisites)
  graph = Array.new(num_courses) { [] }   # b -> [courses that depend on b]
  indegree = Array.new(num_courses, 0)
  prerequisites.each do |course, prereq|
    graph[prereq] << course
    indegree[course] += 1          # course gains one unmet prerequisite
  end

  queue = (0...num_courses).select { |c| indegree[c] == 0 }   # ready courses
  read = 0
  taken = 0
  while read < queue.length
    course = queue[read]; read += 1
    taken += 1                     # we can take this course now
    graph[course].each do |dependent|
      indegree[dependent] -= 1     # one prereq satisfied
      queue << dependent if indegree[dependent] == 0
    end
  end

  taken == num_courses             # all taken => acyclic => finishable
end

    Number of Connected Components (Union-Find) New skill

    Prompt: n nodes labeled 0..n-1 and a list of undirected edges. How many connected components? (LC 323)

    Strategy: Union-Find (DSU). Start with n components. For each edge, union the two endpoints; if they were in different sets, merge and decrement the count. find uses path compression and union uses union by rank for near-O(1) amortized ops. This is the go-to for dynamic connectivity.

    Complexity: Time O((V + E)·α(V)) ≈ O(V + E) · Space O(V).

    Ruby

    # Input:  n = 5, edges = [[0,1],[1,2],[3,4]]   Output: 2

class UnionFind
  def initialize(n)
    @parent = (0...n).to_a         # each node starts as its own root
    @rank   = Array.new(n, 0)      # tree height heuristic
    @count  = n                    # number of components
  end
  attr_reader :count

  def find(x)
    # Path compression: point x straight at the root on the way back.
    @parent[x] = find(@parent[x]) if @parent[x] != x
    @parent[x]
  end

  def union(a, b)
    ra, rb = find(a), find(b)
    return if ra == rb             # already connected; no merge
    # Union by rank: attach the shorter tree under the taller.
    ra, rb = rb, ra if @rank[ra] < @rank[rb]
    @parent[rb] = ra
    @rank[ra] += 1 if @rank[ra] == @rank[rb]
    @count -= 1                    # two components became one
  end
end

def count_components(n, edges)
  uf = UnionFind.new(n)
  edges.each { |a, b| uf.union(a, b) }
  uf.count
end

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))   # each node is its own root
        self.rank = [0] * n
        self.count = n                 # number of components

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # path compression
        return self.parent[x]

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:                   # already connected
            return
        if self.rank[ra] < self.rank[rb]:   # union by rank
            ra, rb = rb, ra
        self.parent[rb] = ra
        if self.rank[ra] == self.rank[rb]:
            self.rank[ra] += 1
        self.count -= 1                # merged two -> one

    uf = UnionFind(n)
    for a, b in edges:
        uf.union(a, b)
    return uf.count

    Backtracking ★ gap-filler — build a candidate, recurse, undo, repeat

    Signals: "find/return all" combinations, permutations, subsets, partitions, or placements; constraint satisfaction (N-Queens, Sudoku). The shape is always the same: choose → explore (recurse) → un-choose. Prune dead branches early. Complexity is exponential by nature — that's expected; show you can prune and that you stop at the answer.

    The universal backtracking skeleton

    def backtrack(state, choices):
    if is_complete(state):          # reached a full candidate
        record(state); return
    for choice in valid_choices(choices, state):
        make(choice, state)         # CHOOSE
        backtrack(state, ...)       # EXPLORE deeper
        undo(choice, state)         # UN-CHOOSE (the "backtrack")

    Every problem below is this skeleton with different is_complete, valid_choices, and pruning. Master the shape once; vary the fill-ins.

    Subsets (power set) New skill

    Prompt: Return all subsets of a distinct-integer array. (LC 78)

    Strategy: At index i, every element is either in or out. Recurse with a start index so we only ever look forward (no duplicate subsets / no reordering). Record the current path at every node — each path is a valid subset.

    Complexity: Time O(n·2ⁿ) · Space O(n) recursion (output is 2ⁿ).

def subsets(nums)
  result = []
  path = []
  backtrack = lambda do |start|
    result << path.dup             # every prefix-path is a subset (copy it!)
    (start...nums.length).each do |i|
      path << nums[i]              # CHOOSE nums[i]
      backtrack.call(i + 1)        # EXPLORE only later elements
      path.pop                     # UN-CHOOSE
    end
  end
  backtrack.call(0)
  result
end

    Permutations New skill

    Prompt: Return all orderings of a distinct-integer array. (LC 46)

    Strategy: Unlike subsets, order matters and each element is used exactly once, so we don't carry a start — instead track which elements are used. When the path length hits n, it's a complete permutation.

    Complexity: Time O(n·n!) · Space O(n).

def permutations(nums)
  result = []
  path = []
  used = Array.new(nums.length, false)
  backtrack = lambda do
    if path.length == nums.length
      result << path.dup           # full-length path = one permutation
      return
    end
    nums.each_with_index do |num, i|
      next if used[i]              # skip elements already in the path
      used[i] = true; path << num  # CHOOSE
      backtrack.call               # EXPLORE
      path.pop; used[i] = false    # UN-CHOOSE
    end
  end
  backtrack.call
  result
end

    Prompt: Given distinct candidates and a target, return all unique combinations summing to target. Each number may be reused unlimited times. (LC 39)

def combination_sum(candidates, target)
  candidates = candidates.sort     # enables the early break below
  result = []
  path = []

    if remaining == 0
      result << path.dup           # exact hit
      return

      break if candidates[i] > remaining   # sorted: nothing further fits
      path << candidates[i]                  # CHOOSE
      backtrack.call(i, remaining - candidates[i])  # reuse allowed => same i
      path.pop                              # UN-CHOOSE
    end
  end
  backtrack.call(0, target)
  result

    Word Search (grid backtracking) New skill

    Prompt: Does word exist in the grid via 4-adjacent moves, each cell used once? (LC 79)

    Strategy: DFS from every cell matching the first letter. The backtracking twist is the used-cell handling: temporarily mark the current cell (overwrite or visited-set), recurse to the 4 neighbors for the next letter, then restore it so other paths can use it. Return on the first success.

    Complexity: Time O(rows·cols·4^len) worst case · Space O(len) recursion.

    Ruby

def exist(board, word)
  rows, cols = board.length, board[0].length

  dfs = lambda do |r, c, idx|
    return true if idx == word.length          # matched every letter
    return false if r < 0 || c < 0 || r >= rows || c >= cols
    return false if board[r][c] != word[idx]   # letter mismatch

    saved = board[r][c]
    board[r][c] = "#"              # mark used so this path can't revisit it
    found = dfs.call(r+1, c, idx+1) || dfs.call(r-1, c, idx+1) ||
            dfs.call(r, c+1, idx+1) || dfs.call(r, c-1, idx+1)
    board[r][c] = saved            # RESTORE for other paths (the backtrack)
    found
  end

  rows.times do |r|
    cols.times do |c|
      return true if dfs.call(r, c, 0)
    end
  end
  false
end

def coin_change(coins, amount)
  big = amount + 1                 # sentinel "infinity" (can't exceed amount coins)
  dp = Array.new(amount + 1, big)
  dp[0] = 0                        # zero coins make amount 0
  (1..amount).each do |a|
    coins.each do |coin|
      # If this coin fits, maybe it improves the count for amount a.
      dp[a] = [dp[a], dp[a - coin] + 1].min if coin <= a
    end
  end
  dp[amount] == big ? -1 : dp[amount]

            if dp[j] and s[j:i] in words:
                dp[i] = True
                break                # one split suffices
    return dp[n]

print(word_break("leetcode", ["leet", "code"]))   # -> True
print(word_break("catsand", ["cats", "dog"]))      # -> False

    Longest Common Subsequence New skill

    Prompt: Length of the longest subsequence common to both strings (chars in order, not necessarily contiguous). (LC 1143)

    Strategy: The canonical 2-D DP. dp[i][j] = LCS length of a[0...i] and b[0...j]. If the last chars match, dp[i][j] = dp[i-1][j-1] + 1; else max(dp[i-1][j], dp[i][j-1]). Row/column 0 are zero (empty string). This grid pattern underlies edit distance, too.

    Complexity: Time O(m·n) · Space O(m·n) (reducible to O(min) with two rows).

    Ruby

    # Input:  a = "abcde", b = "ace"   Output: 3  ("ace")

def longest_common_subsequence(a, b)
  m, n = a.length, b.length
  # dp[i][j] = LCS of a[0...i] and b[0...j]; extra row/col of zeros for empties.
  dp = Array.new(m + 1) { Array.new(n + 1, 0) }
  (1..m).each do |i|
    (1..n).each do |j|
      if a[i - 1] == b[j - 1]
        dp[i][j] = dp[i - 1][j - 1] + 1        # chars match: extend diagonal
      else
        dp[i][j] = [dp[i - 1][j], dp[i][j - 1]].max  # drop one char, take best
      end
    end
  end
  dp[m][n]
end

p longest_common_subsequence("abcde", "ace")   # => 3

    Python

    # Input:  a = "abcde", b = "ace"   Output: 3  ("ace")

def longest_common_subsequence(a, b):
    m, n = len(a), len(b)
    # dp[i][j] = LCS of a[:i] and b[:j]; row/col 0 are zeros (empty string).
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if a[i - 1] == b[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1          # match: diagonal + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])  # best of dropping one
    return dp[m][n]

print(longest_common_subsequence("abcde", "ace"))   # -> 3

    Design / OOD "design a class that supports these operations"

    The whole game: pick the data structure(s) that make each required operation hit its target complexity, then keep them in sync. Most "design X" problems are really "combine a hash map with one other structure (list / heap / doubly-linked list / sorted array)."

    OOD interview approach

    1. List the operations and the target complexity for each ("get and put should both be O(1)").
    2. Clarify: capacity/eviction policy? thread-safe? duplicate keys? what to return on miss?
    3. Pick structures per op, then state the invariant that keeps them consistent.
    4. Code the skeleton (constructor + each method), then walk a small sequence of calls.

    LRU Cache ✓ Reviewed

    Prompt: Fixed-capacity cache with O(1) get and put; evict the least-recently-used key when full. (LC 146)

    Strategy: Need O(1) lookup and O(1) "move to most-recent" / "evict oldest." Canonical answer: hash map + doubly-linked list. But both languages give you an insertion-ordered hash (Ruby Hash, Python OrderedDict), so deleting and re-inserting a key makes it "most recent" in O(1) — much cleaner to code under pressure. Mention the DLL version exists.

    Complexity: get/put O(1) · Space O(capacity).

    Ruby

    # Ruby Hash preserves insertion order, so the FIRST key is the oldest.

class LRUCache
  def initialize(capacity)
    @capacity = capacity
    @store = {}                    # key => value, ordered by recency (oldest first)
  end

  def get(key)
    return -1 unless @store.key?(key)
    value = @store.delete(key)     # remove...
    @store[key] = value            # ...and re-insert => now most-recent
    value
  end

  def put(key, value)
    @store.delete(key) if @store.key?(key)   # refresh position if present
    @store[key] = value
    if @store.size > @capacity
      @store.delete(@store.keys.first)        # evict the oldest (front)
    end
  end
end

c = LRUCache.new(2)
c.put(1, 1); c.put(2, 2)
p c.get(1)        # => 1   (1 is now most-recent)
c.put(3, 3)       # evicts key 2 (least recently used)
p c.get(2)        # => -1

    Python

    from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.store = OrderedDict()     # ordered by recency (oldest first)

    def get(self, key):
        if key not in self.store:
            return -1
        self.store.move_to_end(key)    # mark most-recent in O(1)
        return self.store[key]

    def put(self, key, value):
        if key in self.store:
            self.store.move_to_end(key)
        self.store[key] = value
        if len(self.store) > self.capacity:
            self.store.popitem(last=False)   # evict oldest (front)

c.put(1, 1); c.put(2, 2)
print(c.get(1))   # -> 1
c.put(3, 3)       # evicts key 2
print(c.get(2))   # -> -1

    Logger Rate Limiter ✓ Reviewed

    Prompt: should_print(timestamp, message) returns true only if that message hasn't printed in the last 10 seconds. (LC 359)

    Strategy: Hash of message => next_allowed_timestamp. Print iff timestamp >= next_allowed[message]; when you print, set the next-allowed to timestamp + 10. O(1) per call. (If memory must be bounded, sweep expired entries or use a queue + set — mention it.)

    class Logger

      @next_allowed[message] = timestamp + 10
      true
    else
      false
    end
  end
end

    Browser History ✓ Reviewed

    Prompt: Support visit(url), back(steps), forward(steps). Visiting clears all forward history. (LC 1472)

    Strategy: An array as the history plus a current index is the cleanest model. visit truncates everything after current then appends. back/forward just clamp the index by steps within [0, size-1]. (Two stacks also works; the index version is less error-prone.)

    Complexity: visit O(1) amortized · back/forward O(1) · Space O(n).

    Ruby

    class BrowserHistory
  def initialize(homepage)
    @history = [homepage]
    @current = 0                   # index of the page we're on
  end

  def visit(url)
    @history = @history[0..@current]   # drop forward history
    @history << url
    @current = @history.length - 1
  end

  def back(steps)
    @current = [@current - steps, 0].max         # clamp at the start
    @history[@current]
  end

  def forward(steps)
    @current = [@current + steps, @history.length - 1].min  # clamp at the end
    @history[@current]
  end
end

b = BrowserHistory.new("a.com")
b.visit("b.com"); b.visit("c.com")
p b.back(1)       # => "b.com"
p b.forward(1)    # => "c.com"
b.visit("d.com")  # clears forward
p b.forward(1)    # => "d.com" (nowhere forward to go)

    Python

    class BrowserHistory:
    def __init__(self, homepage):
        self.history = [homepage]
        self.current = 0              # index of current page

    def visit(self, url):
        del self.history[self.current + 1:]   # drop forward history
        self.history.append(url)
        self.current = len(self.history) - 1

    def back(self, steps):
        self.current = max(self.current - steps, 0)
        return self.history[self.current]

    def forward(self, steps):
        self.current = min(self.current + steps, len(self.history) - 1)
        return self.history[self.current]

    Time-Based Key-Value Store New skill

    Prompt: set(key, value, timestamp) and get(key, timestamp) returning the value with the largest stored timestamp ≤ the query (or "" if none). Timestamps for a key are strictly increasing. (LC 981)

    Strategy: Great cross-pattern problem: hash of key => list of [timestamp, value] (already sorted because sets arrive increasing), then a binary search (upper-bound) on get to find the rightmost entry with timestamp ≤ query. Ties together hashing + binary search.

    Complexity: set O(1) · get O(log n) · Space O(total entries).

    Ruby

    class TimeMap
  def initialize
    # key => array of [timestamp, value], appended in increasing timestamp order
    @store = Hash.new { |h, k| h[k] = [] }
  end

  def set(key, value, timestamp)
    @store[key] << [timestamp, value]
  end

  def get(key, timestamp)
    entries = @store[key]
    return "" if entries.empty?
    # Binary search for the rightmost entry with ts <= timestamp.
    lo, hi = 0, entries.length - 1
    result = ""
    while lo <= hi
      mid = lo + (hi - lo) / 2
      if entries[mid][0] <= timestamp
        result = entries[mid][1]   # candidate; look for a later valid one
        lo = mid + 1
      else
        hi = mid - 1               # too new; search earlier
      end
    end
    result
  end
end

tm.set("foo", "bar", 1)
p tm.get("foo", 1)     # => "bar"
p tm.get("foo", 3)     # => "bar"  (latest <= 3)
tm.set("foo", "baz", 4)
p tm.get("foo", 4)     # => "baz"
p tm.get("foo", 0)     # => ""    (nothing <= 0)

    Python

    from collections import defaultdict
import bisect

class TimeMap:
    def __init__(self):
        self.store = defaultdict(list)   # key -> [(timestamp, value), ...]

    def set(self, key, value, timestamp):
        self.store[key].append((timestamp, value))   # increasing order

    def get(self, key, timestamp):
        entries = self.store[key]
        # Find insertion point of (timestamp, "~"): everything left is <= timestamp.
        i = bisect.bisect_right(entries, (timestamp, chr(127)))
        return entries[i - 1][1] if i > 0 else ""
