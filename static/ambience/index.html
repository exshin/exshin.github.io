    What topgrading is (read first)

    A structured interview method (from the book Topgrading) built to get past rehearsed highlight reels. Its signature moves:

    • Chronological, no skipping. Education → first job → … → now. Each chapter gets the same questions, so inconsistencies surface.
    • The "boss" questions are the teeth. "Who was your manager? What would they say about you?" — and later, the recruiter may offer to reference-check, so answer as if they will. This is the anti-BS mechanism: give the real picture.
    • Failures and weaknesses are asked directly, every chapter. Dodging reads worse than the flaw itself.
    • They watch the trend line. Increasing scope, ownership, and self-awareness over time is the signal (their value: slope > y-intercept).

    How to win it (the meta-patterns)

    DO

    • Prepare a tight 60–90s summary per role: dates, what it was, your scope, one win, one failure/lesson, why you left. Rehearse out loud.
    • Be specific and quantified where you can — names, numbers, systems.
    • Give honest, self-aware weaknesses with what you did about them.
    • Keep the trend consistent: each move toward more ownership / harder problems.
    • Answer the "boss" questions candidly — assume references get called.

    DON'T

    • Bad-mouth any employer, boss, or teammate — reframe friction as a problem you helped with.
    • Give fake weaknesses ("I work too hard," "perfectionist") — topgrading is designed to catch exactly this.
    • Be vague on dates or scope — gaps and hand-waving invite digging.
    • Let it become a monologue — it's a dialogue; pause, let them steer.
    • Inflate titles/impact you can't defend under follow-ups or a reference check.

    1 · Transcript → speaker turns

    Prompt. You're given a transcript as a list of tokens {speaker, ts, text} sorted by ts (seconds).

    • Part A: Merge consecutive tokens from the same speaker into a single "turn" with combined text and start/end times.
    • Part B: Even for the same speaker, start a new turn if the gap since their last token exceeds max_gap seconds (a pause = a new thought).
    • Part C: Add a keep filter so we only include certain speakers (e.g. drop a third "nurse" speaker), without breaking merging.
    Ruby — Parts A + B + C
    # Part C handled first by filtering, then A+B in one pass.
def speaker_turns(tokens, max_gap: Float::INFINITY, keep: nil)
  tokens = tokens.select { |t| keep.nil? || keep.include?(t[:speaker]) }
  turns = []
  tokens.each do |t|
    last = turns.last
    same_speaker = last && last[:speaker] == t[:speaker]
    within_gap   = last && (t[:ts] - last[:end]) <= max_gap
    if same_speaker && within_gap
      last[:text] += " " + t[:text]
      last[:end]   = t[:ts]
    else
      turns << { speaker: t[:speaker], text: t[:text].dup, start: t[:ts], end: t[:ts] }
    end
  end
  turns
end

    # Collect all matches with positions, sort longest-first, greedily

def extract_codes_longest(text, dictionary)
  matches = []
  dictionary.each do |phrase, code|
    text.to_enum(:scan, /\b#{Regexp.escape(phrase)}\b/i).each do
      m = Regexp.last_match
      matches << { start: m.begin(0), finish: m.end(0), code: code, len: phrase.length }
    end
  end
  matches.sort_by! { |m| [-m[:len], m[:start]] }   # longest first
  claimed = []                                       # list of [start, finish)
  counts  = Hash.new(0)
  matches.each do |m|
    overlap = claimed.any? { |s, f| m[:start] < f && s < m[:finish] }
    next if overlap
    claimed << [m[:start], m[:finish]]
    counts[m[:code]] += 1
  end
  counts
end

    import re
from collections import defaultdict

def extract_codes(text, dictionary):
    counts = defaultdict(int)
    for phrase, code in dictionary.items():
        pat = re.compile(r"\b" + re.escape(phrase) + r"\b", re.I)
        n = len(pat.findall(text))
        if n:
            counts[code] += n
    return dict(counts)

    matches = []

        for m in re.finditer(r"\b" + re.escape(phrase) + r"\b", text, re.I):
            matches.append((m.start(), m.end(), code, len(phrase)))
    matches.sort(key=lambda x: (-x[3], x[0]))   # longest first
    claimed, counts = [], defaultdict(int)
    for start, finish, code, _ in matches:
        if any(start < f and s < finish for s, f in claimed):
            continue
        claimed.append((start, finish))
        counts[code] += 1
    return dict(counts)

    Talk track: "Part A/B is per-phrase regex with \b boundaries and case-insensitive. Part C is the interesting part: overlapping matches, so I collect spans, sort longest-first, and greedily claim non-overlapping ranges — the same idea as interval scheduling. If the dictionary were huge I'd reach for an Aho-Corasick automaton, but for this scale clarity wins." That last line nails their "good choices, not hyper-optimization" note.

    3 · Merge streaming ASR partials

    Prompt. Ambient transcription emits partial hypotheses as audio arrives, each {start, end, text} (seconds). Later partials can overlap and revise earlier ones.

    • Part A: Given partials sorted by start, merge overlapping/adjacent time ranges into consolidated segments.
    • Part B: When two partials overlap, the later one is the correction — its text wins for the overlapping region. Produce final text per merged segment.
    • Part C: Partials may arrive out of order; handle unsorted input.
    Ruby — Part A + C (merge ranges, sort first)
    # Classic merge-intervals. Part C = just sort up front.
def merge_segments(partials)
  sorted = partials.sort_by { |p| p[:start] }   # Part C
  merged = []
  sorted.each do |p|
    last = merged.last
    if last && p[:start] <= last[:end]          # overlap or touch
      last[:end]  = [last[:end], p[:end]].max
      last[:text] = p[:text]                     # Part B: later text wins
    else
      merged << { start: p[:start], end: p[:end], text: p[:text] }
    end
  end
  merged
end

partials = [
  { start: 0, end: 5,  text: "i have a head" },
  { start: 3, end: 8,  text: "i have a headache" },   # revises the above
  { start: 12, end: 15, text: "for three days" },
]

    def merge_segments(partials):

    4 · Build a SOAP note from a template

    Prompt. Given a template string with {slot} placeholders and a facts hash of extracted values.

    • Part A: Fill each {slot} with its value from facts.
    • Part B: If a slot is missing, leave a clear placeholder like [chief_complaint not documented] instead of crashing.
    • Part C: Support a repeating block {#medications}- {name} {dose}{/medications} over a list of items.
    Ruby — Part A + B
    def fill_template(template, facts)
  template.gsub(/\{(\w+)\}/) do
    key = $1.to_sym
    facts.key?(key) ? facts[key].to_s : "[#{$1} not documented]"
  end
end

fill_template(tpl, chief_complaint: "headache x3 days", vitals: "BP 120/80")
# => "S: headache x3 days\nO: BP 120/80\nA: [assessment not documented]"
    Ruby — Part C (repeating block)
    # Expand repeating sections first, then fill scalar slots.
def render(template, facts)
  expanded = template.gsub(/\{#(\w+)\}(.*?)\{\/\1\}/m) do
    items = facts[$1.to_sym] || []
    inner = $2
    items.map { |item| fill_template(inner, item) }.join
  end
  fill_template(expanded, facts)
end

tpl = "Meds:\n{#medications}- {name} {dose}\n{/medications}"
render(tpl, medications: [{ name: "ibuprofen", dose: "400mg" },
                          { name: "sumatriptan", dose: "50mg" }])
# => "Meds:\n- ibuprofen 400mg\n- sumatriptan 50mg\n"
    Python — Part A + B + C
    import re

def fill_template(template, facts):
    def repl(m):
        key = m.group(1)
        return str(facts[key]) if key in facts else f"[{key} not documented]"
    return re.sub(r"\{(\w+)\}", repl, template)

def render(template, facts):
    def block(m):
        name, inner = m.group(1), m.group(2)
        return "".join(fill_template(inner, item) for item in facts.get(name, []))
    expanded = re.sub(r"\{#(\w+)\}(.*?)\{/\1\}", block, template, flags=re.S)
    return fill_template(expanded, facts)

    Talk track: "A is a regex substitution. B is the real-world bit — clinical notes must not silently drop a missing field, so I surface it explicitly. C I do as a pre-pass that expands repeating blocks, then reuse the scalar filler — composition keeps each function small."     5 · Redact PHI from a transcript

    Prompt. Before logging a transcript we must scrub PHI (protected health info).

    • Part A: Given a list of known names, replace each occurrence (case-insensitive, whole word) with [NAME].
    • Part B: Also redact things that look like an MRN (6+ digits) and dates (MM/DD/YYYY).
    • Part C: Return both the redacted text and a count of redactions per category (for an audit log).
    Ruby — all parts
    def redact(text, names)
  counts = Hash.new(0)

    text = text.gsub(/\b#{Regexp.escape(name)}\b/i) { counts[:name] += 1; "[NAME]" }
  end

end

    Python — all parts
    import re
from collections import defaultdict

def redact(text, names):
    counts = defaultdict(int)

    def sub(pattern, label, key, flags=0):
        nonlocal text
        def repl(_):
            counts[key] += 1
            return label
        text = re.sub(pattern, repl, text, flags=flags)

    for name in names:
        sub(r"\b" + re.escape(name) + r"\b", "[NAME]", "name", re.I)
    sub(r"\b\d{6,}\b", "[MRN]", "mrn")
    sub(r"\b\d{2}/\d{2}/\d{4}\b", "[DATE]", "date")
    return {"text": text, "counts": dict(counts)}

    Grouping / counting / hashmap

    • Top symptoms. From a transcript, return the N most-mentioned symptoms given a symptom vocabulary. Part 2: break ties alphabetically. (Counter + top-K.)

    • Speaker talk-time. From turns with start/end, compute total speaking seconds per speaker and the % split; part 2: who dominated each 5-minute window. (Aggregation + bucketing.)
    • Code frequency report. Given extracted ICD codes across many notes, return count per code and per category prefix (e.g. all R* together). (Group by key + group by derived key.)

    Intervals / merging / overlap

    • Overlapping speech / crosstalk. Given each speaker's speaking intervals, find time ranges where ≥2 people talk at once. (Sweep line / merge.)

    • Free-slot finder. Given a clinician's booked appointment intervals in a day, return open gaps ≥ 15 min. (Sort + gap scan.)
    • Visit sessionizer. Given timestamped events, split into distinct "visits" whenever the gap between consecutive events exceeds T. (Gap-based grouping — same engine as Problem 1B.)

    String matching / regex / parsing

    • Expand abbreviations. Given a map ("pt"→"patient", "hx"→"history"), expand whole-word, case-preserving; part 2: don't expand inside an already-expanded word. (Boundary regex.)
    • Extract structured vitals. Parse "BP 120/80, HR 72, T 98.6" into a hash; part 2: tolerate missing/extra spaces and units. (Regex capture groups.)
    • Dosage parser. From "ibuprofen 400 mg twice daily," extract {drug, amount, unit, frequency}. (Tokenize + pattern.)
    • Highlight matches. Wrap every dictionary phrase occurrence in a note with **…**, longest-match-wins, no nested/overlapping highlights. (Find spans + greedy claim — Problem 2C's engine, different output.)

    Greedy / packing / two-pointer / sorting

    • Chunk a transcript for an LLM. Split turns into chunks under a max char/token budget without splitting a turn; part 2: add K turns of overlap between chunks for context. (Greedy packing + sliding overlap.)
    • Reconcile med lists. Given a prior and a new med list, produce added / removed / unchanged / changed-dose; part 2: match by normalized name. (Set/dict diff on a key.)
    • Merge two ranked code suggestions. Two sorted-by-confidence lists of code suggestions → one deduped list, highest confidence kept per code. (Two-pointer merge / dict max.)
    • Batch by token budget. Given notes with sizes, pack them into the fewest API batches each under a size cap; report batch count. (Greedy bin-fill — note it's a heuristic, not optimal.)

    Lookup / validation / edge-rules

    • Vitals out-of-range flag. Given readings and reference ranges per metric, return which are abnormal and by how much; part 2: age- or sex-specific ranges. (Map lookup + inclusive/exclusive edges.)
    • Pending orders queue. Group extracted action items by priority then insertion order; part 2: cap each bucket at K and report what was dropped. (Bucketing + stable order + log truncation.)
    • Required-field linter. Given a note's filled sections and a per-visit-type required-section list, return missing required sections. (Set difference by config.)
    • Dedup near-identical note lines. Collapse consecutive lines identical after lowercase/trim; part 2: collapse if they differ only by trailing punctuation. (Normalization + run-collapse.)

    State / streaming (a small step up)

    • Live note assembler. Process a stream of events (utterance, correction, finalize_section) and maintain current note state; return the note when finalized. (Reduce over events / simple state machine.)
    • Debounce duplicate codes. A stream emits code suggestions with timestamps; suppress the same code if seen within the last W seconds; part 2: keep the highest-confidence one in the window. (Sliding window + last-seen map.)
    • Idempotent ingestion. Transcript chunks may be delivered more than once (each has an id); process each exactly once and return the assembled transcript. (Seen-set dedup — echoes your Zuora idempotency story.)
